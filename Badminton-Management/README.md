# ระบบจัดการแข่งขันแบดมินตัน

โปรเจกต์นี้เป็นระบบจัดการแข่งขันแบดมินตันชนิดง่าย ๆ ที่ทำงานบนพื้นฐาน HTML (และ JavaScript) เพื่อให้ผู้ใช้งานสามารถจัดคู่แข่งขัน กำหนดสนาม และติดตามผลการแข่งขันได้อย่างสะดวก โดยออกแบบให้ **ใช้งานง่าย (user friendly)** และ **มีหน้าตาสวยงาม** เพื่อให้ทุกคนสามารถจัดการได้ทันทีแม้ไม่เชี่ยวชาญไอที

## คุณสมบัติหลัก

1. **เพิ่มชื่อผู้แข่งขัน**
   - ผู้ใช้งานสามารถเพิ่มชื่อผู้เล่นได้เอง
   - รองรับทั้งการแข่งขันแบบเดี่ยวและแบบคู่
   - จำนวนผู้เล่นไม่จำกัด
   - หากจำนวนไม่ลงตัว (เช่น คู่เดี่ยวจำนวนคี่ หรือคู่จำนวนผู้เล่นไม่ใช่ 2n) ระบบจะสร้างทีมพิเศษ 3 คนให้หนึ่งทีมเพื่อให้การจัดคู่ยังดำเนินได้

2. **กำหนดจำนวนสนาม**
   - ระบุได้ว่ามีสนามกี่สนามสำหรับการแข่งขัน
   - ระบบจะใช้ข้อมูลนี้ในการจัดตารางเพื่อให้คู่แข่งขันลงสนามพร้อมกัน

3. **โหมดแข่งขัน**
   - เลือกได้ระหว่างการแข่งขันแบบเดี่ยว (Single) หรือแบบคู่ (Double)
   - โหมดนี้มีผลต่อการจัดลำดับและการจับคู่

4. **ดูตารางการแข่งขัน**
   - หน้าจอแสดงว่าคู่ไหนแข่งขันกับใคร
   - มีการแสดงลำดับคู่ถัดไป (ถ้ามี) เพื่อให้ผู้จัดหรือผู้ชมทราบว่าใครเจอกันในรอบต่อไป
   - สามารถเลื่อนดูตารางไปเรื่อย ๆ ตามรอบต่าง ๆ ได้

5. **บันทึกคะแนน**
   - ใส่คะแนนการแข่งขันสำหรับแต่ละคู่ได้
   - คะแนนจะถูกนำไปคำนวณجمعเป็นตารางคะแนนสะสม
   - รองรับการบันทึกคะแนนแต่ละเกม/เซ็ตตามกติกาที่กำหนด

6. **ตารางคะแนนสะสม**
   - ระบบเก็บข้อมูลคะแนนสะสมของผู้เล่น/ทีม
   - แสดงอันดับตามคะแนนที่ได้

## ภาพรวมการใช้งาน

1. เปิดหน้า HTML หลัก (เช่น `index.html`) ในเว็บเบราว์เซอร์
2. เพิ่มผู้เล่นหรือทีมตามต้องการ
3. เลือกจำนวนสนามและโหมดการแข่ง
4. ให้ระบบจัดคู่แข่งขันอัตโนมัติ
5. ดูตารางการแข่งขันที่แสดงคู่ปัจจุบันและคู่ถัดไป
6. ป้อนคะแนนเมื่อการแข่งขันจบ
7. ดูผลสรุปและตารางคะแนนสะสม

## โครงสร้างไฟล์แนะนำ

```
Badminton-Management/
├── index.html        # หน้าเว็บหลัก (UI และโครงสร้าง) – ใช้ TailwindCSS จาก CDN
├── script.js         # โค้ด JavaScript จัดการสถานะ, ตาราง, และคะแนน
└── README.md         # เอกสารนี้

(style.css ถูกยกเลิกหรือเก็บไว้ว่างๆ ถ้าไม่จำเป็น)
```

## การพัฒนาเพิ่มเติม

- ปรับปรุง UI ให้ใช้งานได้สะดวกบนมือถือ (ใช้ Tailwind เรียบร้อยพร้อมเลย์เอาต์ทันสมัยและไอคอน)
- รองรับการบันทึกข้อมูลลงฐานข้อมูลหรือ localStorage
- ขยายระบบให้รองรับการจัดทัวร์นาเมนต์หรือรอบน็อกเอาต์
- เก็บ **ประวัติอันดับคะแนนรายวัน** พร้อมการดูย้อนหลังของตารางคะแนน
- ออกแบบให้ระบบและหน้าเว็บ **responsive** รองรับการปรับขนาดอัตโนมัติทุกอุปกรณ์
- รองรับการดูผลและตารางคะแนนแบบ **เรียลไทม์** โดยบุคคลทั่วไปสามารถเข้าชมได้

### แนวทาง UX/UI ปรับปรุงล่าสุด
- ปุ่มถูกปิดใช้งานจนกว่าจะเพิ่มผู้เล่นอย่างน้อยสองคน
- ฟิลด์เพิ่มผู้เล่นอัตโนมัติโฟกัสหลังการเพิ่ม
- รายการผู้เล่นแสดงใน card พร้อมปุ่มลบไอคอน
- ใช้โทนสีสบายตา, เงา และเลย์เอาต์สองคอลัมน์บนหน้าจอใหญ่
- ไม่ต้องแก้ไข `style.css` เพราะใช้ Tailwind จาก CDN
